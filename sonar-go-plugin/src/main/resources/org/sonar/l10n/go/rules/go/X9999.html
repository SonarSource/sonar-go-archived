When using functions of the 'unsafe' package in Go, it's up to the developer to make sure the size of the buffer to be written to is large enough to avoid buffer overflows. Buffer overflows can cause the program to crash at a minimum. At worst, a carefully crafted overflow can cause malicious code to be executed.

Packages that import unsafe may be non-portable and are not protected by the Go 1 compatibility guidelines. Go team reserves the right to make changes to the implementation that may break such programs.

Noncompliant Code Example

	import (
		"fmt"
		"unsafe"
	)

Compliant Solution

	import (
		"fmt"
	)
